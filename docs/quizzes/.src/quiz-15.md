# 📋 Quiz 15: Algoritmos e Fluxogramas

Teste sua capacidade de planejar soluções lógicas e entender a linguagem visual dos algoritmos.

1. Qual a definição mais precisa de um "Algoritmo"?
    - [ ] Uma linguagem de programação complexa
    - [x] Uma sequência finita de passos claros para resolver um problema
    - [ ] Um erro que ocorre no sensor do computador
    - [ ] O nome do criador do primeiro computador
    *feedback: Receitas, manuais de montagem e GPS são exemplos de algoritmos.*

2. Por que dizemos que um algoritmo deve ser "finito"?
    - [ ] Porque deve caber em uma folha de papel
    - [ ] Porque deve ser escrito em letras pequenas
    - [x] Porque ele deve ter um fim obrigatoriamente, após executar seus passos
    - [ ] Porque só funciona em números inteiros
    *feedback: Um algoritmo que entra em um "loop infinito" é considerado um erro grave.*

3. No padrão visual de fluxogramas, qual símbolo representa o **Início** ou o **Fim** do processo?
    - [ ] Retângulo
    - [ ] Losango
    - [x] Oval (ou Elipse)
    - [ ] Círculo
    *feedback: O símbolo oval delimita as fronteiras de onde o algoritmo começa e termina.*

4. Qual símbolo é utilizado para indicar uma **Ação** ou **Processo** (como um cálculo ou movimento de dados)?
    - [x] Retângulo
    - [ ] Losango
    - [ ] Triângulo
    - [ ] Trapézio
    *feedback: O retângulo é usado para instruções diretas do tipo "Faça isso".*

5. O símbolo em formato de **Losango** é fundamental para representar:
    - [ ] Uma soma matemática
    - [x] Uma Decisão ou Pergunta (Desvio Condicional)
    - [ ] A entrada de dados via teclado
    - [ ] A impressão de um relatório
    *feedback: O losango sempre gera duas saídas: uma para "Sim" (Verdadeiro) e outra para "Não" (Falso).*

6. A estrutura lógica "SE (condição) ENTÃO (ação) SENÃO (outra ação)" é conhecida como:
    - [ ] Repetição
    - [x] Seleção ou Desvio Condicional
    - [ ] Conector Lógico
    - [ ] Procedimento de Busca
    *feedback: É a base da tomada de decisão em qualquer linguagem de programação.*

7. No fluxograma, a direção do fluxo de execução é indicada por:
    - [ ] Cores diferentes
    - [ ] O tamanho dos símbolos
    - [x] Setas de conexão
    - [ ] Números de série
    *feedback: As setas guiam o olhar e o processamento através do caminho planejado.*

8. O que é "Pseudocódigo" (ou Portugol)?
    - [ ] Um código escrito em japonês
    - [x] Uma forma de escrever algoritmos usando linguagem humana estruturada
    - [ ] Um vírus que imita programas reais
    - [ ] O código que a CPU executa diretamente
    *feedback: Ele serve para testar a lógica antes de "traduzir" para uma linguagem de programação real.*

9. Qual a principal vantagem de desenhar um fluxograma antes de começar a codificar um programa?
    - [ ] Deixa o programa mais colorido
    - [ ] Faz o computador gastar menos energia
    - [x] Ajuda a visualizar erros de lógica e planejar o fluxo correto do sistema
    - [ ] É obrigatório por lei
    *feedback: Planejar evita retrabalho e códigos confusos (conhecidos como "código espaguete").*

10. Se em um fluxograma de decisão a pergunta for "O saldo é maior que 100?" e o valor real for 50, o fluxo deve seguir pelo caminho:
    - [ ] Sim (True)
    - [x] Não (False)
    - [ ] Ambos os caminhos ao mesmo tempo
    - [ ] O fluxo para e gera um erro
    *feedback: Se a condição é falsa, o algoritmo deve seguir obrigatoriamente o caminho do "Não" (Else).*

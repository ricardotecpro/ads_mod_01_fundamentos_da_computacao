## 🧩 O que é Programar?

Programar é o ato de dar instruções precisas a um computador para que ele execute uma tarefa.

É traduzir a lógica humana (Algoritmos) para uma linguagem que a máquina entenda. { .fragment }

---

## 🏗️ Níveis de Linguagem

1. **Baixo Nível**: Próximo do hardware (Binário, Assembly). Difícil para humanos, ultraveloz para a CPU. { .fragment }
2. **Médio Nível**: C e C++. Equilíbrio entre velocidade e legibilidade. { .fragment }
3. **Alto Nível**: Python, JS, Java. Muito parecido com inglês, focado na produtividade. { .fragment }

---

## 🔄 A Jornada do Código

Como o texto que você escreve vira ação?

1. **Código Fonte**: O texto que o programador escreve. { .fragment }
2. **Tradução**: Um software (Compilador ou Interpretador) lê o texto. { .fragment }
3. **Código de Máquina**: O resultado final são os 0s e 1s que a CPU executa. { .fragment }

---

## ⚙️ Compilado vs Interpretado

- **Compilada (Ex: C, Rust)**: O código é traduzido uma única vez em um arquivo `.exe`. Super rápido. { .fragment }
- **Interpretada (Ex: Python, JS)**: O código é lido linha por linha durante a execução. Mais flexível e fácil de testar. { .fragment }

---

## 🧱 Bloco 1: Variáveis

Pense na variável como uma **gaveta** identificada na RAM.

- Você dá um **nome** à gaveta (Ex: `idade`). { .fragment }
- Você guarda um **valor** nela (Ex: `25`). { .fragment }
- Você pode trocar o conteúdo a qualquer momento. { .fragment }

---

## 📦 Bloco 2: Tipos de Dados

O computador precisa saber o que tem na gaveta para calcular certo:

- **int**: Números inteiros (10, -5). { .fragment }
- **float**: Números decimais (1.75). { .fragment }
- **str**: Texto (Strings - "Olá"). { .fragment }
- **bool**: Lógica (True ou False). { .fragment }

---

## 🚦 Bloco 3: Estruturas de Controle

- **IF (Se)**: Desvios Condicionais. { .fragment }
- **FOR / WHILE (Laços)**: Repetições automáticas. { .fragment }
- Sem essas estruturas, o programa seria apenas uma lista linear e burra. { .fragment }

---

## 🛠️ Bloco 4: Funções

Uma função é um bloco de código que você "empacota" para reutilizar depois.

Em vez de escrever 10 linhas para "Enviar E-mail" toda vez, você cria a função `enviar_email()` e a chama pelo nome. { .fragment }

---

## 🌍 Olá Mundo! (Hello World)

A primeira tradição de todo programador.

```python
# Em Python é simples assim:
print("Olá, Mundo!")
```

Parece pouco, mas aqui você já usou: uma função (`print`) e uma String. { .fragment }

---

## 🐛 O Que é um Bug?

O nome vem de um besouro real que travou um computador valvulado nos anos 40.

Hoje, um bug é um erro na lógica ou na sintaxe do código que faz o programa se comportar de forma inesperada. { .fragment }
Depurar (Debug) é o ato de caçar esses erros. { .fragment }

---

## 🎨 Clean Code (Código Limpo)

"Código é feito para humanos lerem, e ocasionalmente para máquinas executarem."

- Use nomes claros: `preco_final` em vez de `pf`. { .fragment }
- Comente o "porquê", não o "quê". { .fragment }
- Organize o código de forma lógica. { .fragment }

---

## ✍️ Exercício Rápido 1

Qual tipo de dado deve ser usado para guardar o preço de um produto: `int` ou `float`?

- Resposta: **float** (pois preços têm centavos/decimais). { .fragment }

---

## ✍️ Exercício Rápido 2

Qual a diferença entre uma linguagem de Alto Nível e uma de Baixo Nível?

- Resposta: Alto nível é mais fácil para humanos; baixo nível é mais próximo do hardware. { .fragment }

---

## 🏗️ Desafio: O Loop Infinito

O que acontece se um algoritmo de repetição (WHILE) nunca encontrar sua condição de parada?

- O programa entra em loop infinito, consumindo 100% da CPU e parando de responder (travando). { .fragment }

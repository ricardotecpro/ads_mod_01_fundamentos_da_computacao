## 🧩 O que é um Algoritmo?

Não é algo exclusivo da informática.

Um algoritmo é uma **sequência finita de passos** para realizar uma tarefa ou resolver um problema. { .fragment }
- Ex: Receita de bolo, manual do IKEA, GPS. { .fragment }

---

## 🏗️ As 3 Regras de Ouro

Um algoritmo precisa ser:

1. **Finito**: Deve ter um fim. { .fragment }
2. **Definido**: Cada passo deve ser claro e sem ambiguidade. { .fragment }
3. **Eficaz**: Deve realmente resolver o problema. { .fragment }

---

## 💡 Pensamento Computacional

Antes de programar, precisamos:
- **Decomposição**: Dividir um problemão em probleminhas. { .fragment }
- **Abstração**: Focar no que é importante e ignorar o resto. { .fragment }
- **Padrões**: Identificar o que se repete. { .fragment }

---

## 📊 O que é um Fluxograma?

É a representação **visual** de um algoritmo.

- Facilita a compreensão do fluxo lógico. { .fragment }
- Ajuda a encontrar erros antes de escrever o código. { .fragment }

---

## 🛑 Símbolos: Início e Fim

- **Formato**: Oval (ou Elipse). { .fragment }
- Todo fluxograma DEVE ter um ponto de partida e um ponto de chegada. { .fragment }

---

## ⚙️ Símbolos: Processo

- **Formato**: Retângulo. { .fragment }
- Representa uma ação, cálculo ou atribuição. { .fragment }
- Ex: `X = A + B` ou `Fritar ovo`. { .fragment }

---

## ❓ Símbolos: Decisão

- **Formato**: Losango (Diamante). { .fragment }
- Representa uma pergunta que só tem duas respostas: **SIM** ou **NÃO**. { .fragment }
- Gera dois caminhos diferentes no fluxo. { .fragment }

---

## 📥 Símbolos: Entrada/Saída

- **Formato**: Paralelogramo. { .fragment }
- Representa a leitura de um dado ou a exibição de um resultado. { .fragment }
- Ex: `Ler Idade` ou `Mostrar "Olá!"`. { .fragment }

---

## 📝 Exemplo: Atravessar a Rua

1. Olhar para a esquerda e direita. { .fragment }
2. **O carro está vindo?** (Decisão) { .fragment }
   - **Sim**: Esperar e voltar ao passo 1. { .fragment }
   - **Não**: Atravessar. { .fragment }

---

## 🔄 Estruturas de Seleção (SE)

A base de toda lógica inteligente.

`SE (condição) ENTÃO { ação A } SENÃO { ação B }` { .fragment }

Ex: `SE idade >= 18 ENTÃO "Pode dirigir" SENÃO "Não pode"`. { .fragment }

---

## 🧪 Exemplo: Fazer Café

1. Pegar café.
2. **A água está quente?** (Decisão) { .fragment }
   - NÃO: Esquentar água. { .fragment }
3. Coar café. { .fragment }
4. Beber! { .fragment }

---

## ✍️ Exercício Rápido 1

Qual símbolo é usado para representar uma decisão?

- Resposta: **Losango**. { .fragment }

---

## ✍️ Exercício Rápido 2

Algoritmos podem ser infinitos?

- Resposta: **Não**. Eles devem ser finitos e chegar a um resultado. { .fragment }

---

## 🏗️ Desafio: O Semáforo

Como seria o fluxograma de um semáforo inteligente?

- Se `Carro_Detectado AND Luz_Vermelha_Ha_3_minutos`
- ENTÃO `Mudar para Verde`. { .fragment }

---

## 🚦 Por que Fluxogramas?

"Desenhar um fluxograma é como fazer o rascunho de uma redação."

- Evita o "Espaguete": códigos confusos e sem direção. { .fragment }
- É uma linguagem universal entre programadores. { .fragment }

---

## 📊 Resumo da Aula

1. Algoritmos são sequências lógicas de passos. { .fragment }
2. Fluxogramas usam símbolos padronizados para visualização. { .fragment }
3. Decisões (Losangos) criam caminhos alternativos. { .fragment }
4. Planejar a lógica economiza horas de depuração. { .fragment }
